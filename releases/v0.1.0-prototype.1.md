# SDD Master v0.1.0-prototype.1

## Status

Prototype prerelease publicado no npm.

## Objetivo

Alinhar a versão do pacote, documentação pública e tag futura ao estado atual do projeto após a tag inicial `v0.1.0-prototype`, sem mover ou reescrever histórico Git.

## Escopo

- Versão npm preparada: `0.1.0-prototype.1`.
- Tag Git planejada: `v0.1.0-prototype.1`.
- Tag npm configurada: `prototype`.
- GitHub Release final: não publicada.
- npm publish: realizado.

## npm

Publicado no npm:

```bash
npm install -g sdd-master@prototype
```

Versão publicada:

```text
sdd-master@0.1.0-prototype.1
```

Dist-tag:

```text
prototype
```

Como esta é a primeira publicação do pacote, o npm também manteve `latest` apontando para `0.1.0-prototype.1`. O registry bloqueou a remoção de `latest` enquanto essa é a única versão publicada.

A versão continua sendo prototype e não deve ser tratada como release estável. Enquanto não houver release estável, use:

```bash
npm install -g sdd-master@prototype
```

Evite usar:

```bash
npm install -g sdd-master
```

## Segurança

- Nenhum `.env` real deve entrar no commit, pacote ou release.
- Nenhum segredo, token, credencial, chave privada ou dado sensível deve ser incluído.
- `.sdd-master/` da raiz do pacote permanece ausente.
- PDFs locais untracked permanecem fora do commit e do pacote.

## Validações esperadas

```bash
npm run check
npm run release:check
npm publish --dry-run --access public --tag prototype
sdd master git --pre-push
```
